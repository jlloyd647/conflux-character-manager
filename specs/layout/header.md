# [Header]

## Purpose

Exists to give the user access to commonly used links and menus that are viewable across all pages. 

## Responsibilities

- Manages routes and route information
- Displays links based on user type
- Displays a menu and dropdown
- Displays a logo that acts as a home page link

---

## Does Not Own

- User data

---

## Relationships

- routes.md
- layout.md

---

## Rules

- header must span the width of the application
- logo must be located on the left of the header
- user dropdown menu must be located on the right of the header
- route links must be located to the left of the user dropdown menu
- header must give a small buffer of space between elements within the header and the edges of the header
- header must be 10% of the screen height
- header must be at the top of the page regardless of route
