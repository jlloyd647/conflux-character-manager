# [Layout]

## Purpose

This creates the rules for how the major components are layed out throughout the application. This will define site wide features that will be visible regardless of the current endpoint the user is on.

## Responsibilities

- Orientation and layout of whole components

---

## Does Not Own

- Individual components and their features

---

## Relationships

- main-view.md
- header.md
- footer.md
- notifications.md

---

## Rules

- The layout does not make changes within the components, it only displays relevant components
